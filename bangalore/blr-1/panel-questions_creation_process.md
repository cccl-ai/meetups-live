## Design System

The site uses a shared Jekyll layout and CSS. Key CSS classes available for use in markdown pages:

- `.speaker-card` ‚Äî dark card with border, used for each question
- `.talk-title` ‚Äî italic text inside a card (used for question text)
- `.speaker-links` ‚Äî small footer row inside a card (used for ID + name)
- `.highlight-box` ‚Äî gold-bordered callout box
- `.people-pills` ‚Äî inline tag-style list of names

The design uses three fonts: Instrument Serif (headings), DM Sans (body), JetBrains Mono (code/IDs).

## How the Panel Questions Page Works

The page is a standard Jekyll markdown file with frontmatter. The layout renders `page.title`, `page.event_date`, and `page.meta` in a styled header. Body content is rendered as HTML via kramdown.

### Frontmatter fields

```yaml
---
layout: default
title: "CCCL Bangalore #1 ‚Äî Panel Questions"
description: "..."
event_date: "25 April 2026"
meta:
  - label: Questions
    value: "23 submitted"
  - label: Contributors
    value: "10 community members"
  - label: Location
    value: "Bangalore"
footer_links:                          # overrides the default London footer
  - label: WhatsApp
    url: https://cccl.ai/blr/whatsapp
  - label: Slack
    url: https://cccl.ai/slack
  - label: Events
    url: https://cccl.ai/blr/luma
  - label: cccl.ai
    url: https://cccl.ai
---
```

### `footer_links` ‚Äî how it works

The default footer in `_layouts/default.html` points to London URLs. A `footer_links` frontmatter key overrides it for any page that specifies it. This was added specifically for Bangalore so London pages are unaffected.

The Liquid template in `default.html`:
```liquid
{% if page.footer_links %}
  {% for link in page.footer_links %}{% unless forloop.first %} ¬∑ {% endunless %}<a href="{{ link.url }}">{{ link.label }}</a>{% endfor %}
{% else %}
  <!-- London defaults -->
{% endif %}
```

### Question card format

Each question is an HTML div block inside the markdown:

```html
<div class="speaker-card">
<div class="talk-title">verbatim question text here</div>
<div class="speaker-links"><code>questionID</code> <span>‚Äî Contributor Name</span></div>
</div>
```

Questions are verbatim from the source. Do not paraphrase.

## Jekyll ‚Äî How to Run Locally

System Ruby (2.6) is too old. Use homebrew Ruby 4:

```
PATH="/opt/homebrew/opt/ruby/bin:$PATH" GEM_HOME="$HOME/.gem/ruby/4.0.0" bundle exec jekyll serve --livereload
```

First-time setup (already done on this Mac):
```
brew install ruby
PATH="/opt/homebrew/opt/ruby/bin:$PATH" GEM_HOME="$HOME/.gem/ruby/4.0.0" bundle install
```

## How to Extend This

**Add a new question:** Copy a `speaker-card` block, paste verbatim text, update ID and name.

**Add a new theme section:** Add an `## H2 heading` between `---` HR lines, then add cards under it.

**Create pages for future Bangalore events:** Follow the same pattern ‚Äî `bangalore/blr-2/`, etc. Copy the frontmatter including `footer_links`.

**Publish:** `git add` + `git commit` + `git push` to `main`. GitHub Pages deploys automatically (usually within 1‚Äì2 minutes).

---

## Gotchas

- **Do not use system Ruby** ‚Äî it's 2.6, too old for bundler 2.6.9+. Always use the homebrew Ruby PATH prefix.
- **Questions must be verbatim** ‚Äî the user explicitly corrected a paraphrased version. Keep original spelling, capitalisation, and punctuation.
- **footer_links must be set** for every Bangalore page ‚Äî otherwise the footer defaults to London WhatsApp/events links.
- **The repo is under `cccl-ai` GitHub org**, not Vikram's personal account. Push requires SSH access configured for that org.
